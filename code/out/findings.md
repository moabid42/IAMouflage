# GCP Detection-Gap Findings

Generated by `graph/run_queries.py` over the Neo4j graph built from four GCP
detection corpora (Sigma, Elastic, Google SecOps, Panther) joined to the
hacktricks-cloud GCP technique corpus on the canonical IAM permission. Every
finding is a **pure graph query** — no ML model is involved.

## 00_coverage_overview — Detection coverage overview

> Baseline — of every abusable GCP technique in the corpus, how does the merged detection set (Sigma + Elastic + Google SecOps + Panther) classify each? A technique is DETECTED only if one operation it must perform is (a) matched by a rule group on a required permission AND (b) written to a log on by default. The rest split into: CORRELATION_ONLY only a threshold / multi-stage rule fires (needs repetition or a full chain); a single execution stays silent RULE_GAP    (Class A) an operation IS logged by default, but no rule matches it TELEMETRY_GAP (Class B) every observable operation is a Data Access log, OFF by default -> no signature can fire until logging is reconfigured
